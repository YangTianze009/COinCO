# Context Reasoning for Out-of-Context Object Detection via SFT on VLMs

This project fine-tunes **Qwen2.5-VL-3B-Instruct** to detect out-of-context objects in images using three contextual criteria: **Co-occurrence**, **Location**, and **Size**. Training data is generated by multi-model consensus (InternVL, Molmo, Qwen) and used to produce specialized LoRA adapters.

## Prerequisites

### Software
```bash
conda create -n qwen25_3B python=3.10
conda activate qwen25_3B
pip install torch torchvision transformers peft accelerate bitsandbytes
pip install pandas scikit-learn matplotlib pycocotools webdataset tqdm
```

For data generation with InternVL, see `data_generation/internvl_environment.yaml`.

### Configuration

Edit `config.py` to set paths for your environment:
- `DATA_ROOT` — root directory containing COCO data, inpainted images, etc.
- `WEIGHTS_ROOT` — directory containing downloaded model weights

## Pipeline: Step by Step

### Step 1: Data Generation (Context Reasoning Labels)

Generate context reasoning responses from three large VLMs on the inpainted images:

```bash
# Run each model (requires respective model weights)
python data_generation/context_reasoning_InternVL-prompt_3_Part1.py
python data_generation/context_reasoning_InternVL-prompt_3_Part2.py
python data_generation/context_reasoning_Molmo-prompt_3_Part1.py
python data_generation/context_reasoning_Molmo-prompt_3_Part2.py
python data_generation/context_reasoning_Qwen-prompt_3-Part1.py
python data_generation/context_reasoning_Qwen-prompt_3-Part2.py
```

Then combine and analyze results:
- Run `data_generation/context_reasoning-res/*_read_combine_csv.ipynb` to merge Part1+Part2 for each model
- Run `data_generation/context_reasoning-res/Analysis/Analysis.ipynb` to find multi-model agreement samples

The final combined outputs are:
- `context_reasoning-res/internvl/context_reasoning_internvl.csv`
- `context_reasoning-res/molmo/context_reasoning_molmo.csv`
- `context_reasoning-res/qwen/context_reasoning_qwen.csv`
- `context_reasoning-res/qwen/context_reasoning_qwen_testing.csv`

**Or** skip this step entirely and download the identical pre-generated CSVs from [HuggingFace](https://huggingface.co/datasets/COinCO/COinCO-dataset/tree/main/context_reasoning). These are the same files produced by the scripts above.

### Step 2: Build WebDatasets (SFT Training Data)

Convert agreed samples into WebDataset format (one notebook per bias type + split):
- `data_generation/context_reasoning-res/SFT_data/Co_occurance-WebDataset.ipynb` (train)
- `data_generation/context_reasoning-res/SFT_data/co_occurrence-WebDataset-Test.ipynb` (test)
- Similarly for `Location-WebDataset.ipynb`, `size-WebDataset.ipynb`, and their `-Test` variants

### Step 3: Prepare Cached Datasets

For each bias type, run the dataset preparation notebooks:
```bash
# In each of: co_occurrence/, location/, size/
jupyter nbconvert --execute prepare_dataset.ipynb      # Train/eval split
jupyter nbconvert --execute prepare_test_dataset.ipynb  # Test set
```

This creates `cached_dataset/` directories with train/eval/test splits.

### Step 4: Fine-Tune (LoRA on Qwen2.5-VL-3B)

```bash
# For each bias type:
cd co_occurrence/train && bash launch_training.sh
cd location/train && bash launch_training.sh
cd size/train && bash launch_training.sh
```

Each produces a `final_model/` directory containing the LoRA adapter weights.

**Or** skip training and download the pre-trained adapters from [HuggingFace](https://huggingface.co/COinCO/Context_Classification_Models). See each `train/README.md` for download instructions.

### Step 5: Evaluate on Inpainted Images

```bash
# Baseline (original Qwen2.5-VL-3B)
python co_occurrence/eval_inpainted_test_set/baseline/parallel_inference.py

# Fine-tuned (with LoRA adapter)
python co_occurrence/eval_inpainted_test_set/finetuned/parallel_inference.py

# Repeat for location/ and size/
# Also available: eval_inpainted_train_split/ for train-split evaluation
```

Outputs: `parallel_inference_results.csv` with columns `predicted_label, ground_truth_label, predicted_response, ground_truth_response`.

### Step 6: Shortcut Learning Test (Real COCO Images)

Tests whether the model relies on visual artifacts by evaluating on **unmodified** COCO images (all ground truth = In-context). Higher accuracy = less shortcut learning.

```bash
# Baseline
python co_occurrence/eval_real_coco/inference_baseline.py
python location/eval_real_coco/inference_baseline.py
python size/eval_real_coco/inference_baseline.py

# Fine-tuned
python co_occurrence/eval_real_coco/inference_finetuned.py
python location/eval_real_coco/inference_finetuned.py
python size/eval_real_coco/inference_finetuned.py
```

Outputs metrics (Accuracy, Precision, Recall, F1) to stdout. Use `tee` to save logs:
```bash
python inference_baseline.py 2>&1 | tee run_output_baseline.log
```
